This site collects a small amount of information to keep the portfolio working, understand basic traffic, and support signed-in comments.

## Server logs

The web server records standard request logs, including IP address, timestamp, requested URL, user agent, referrer, and response status. These logs are used for security, troubleshooting, uptime checks, traffic summaries, and abuse investigation.

## Comments

Comments require Google sign-in. When you sign in to comment, the site stores the minimum account information needed to show and moderate comments:

- Google account identifier
- email address and verification status
- display name
- profile picture URL, if Google provides one
- submitted comment text
- page URL, timestamp, IP address, and user agent for moderation and abuse prevention

New comments are held for moderation before they appear publicly. Approved comments may show your display name and comment text on the page where you submitted them.

## Analytics

The site may use first-party traffic summaries and server-side logs to understand visits, errors, referrers, and basic usage patterns. These summaries are kept separate from other sites and are used for operations and portfolio due diligence.

## Data sharing

Personal comment data is not sold. If a site or asset is evaluated for sale, the default disclosure package should use aggregate or anonymized traffic evidence rather than personal account records.

## Requests

To ask for comment removal or a privacy-related correction, use the [contact page](/contact/).
